**Version**: 1.0
**Date**: 2026-03-24
**Author**: Sarah (Product Owner)

---

## Executive Summary

Typeless-AI 是一款面向个人及小团队（2-10人）的 AI 驱动语音听写桌面工具，基于 Electron 框架构建，支持 Windows 和 iOS/Android 平台。产品核心目标是提供比现有工具（如 Typeless）更优质的中文语音识别体验，让用户能够通过语音完全替代日常手动打字场景，大幅提升工作效率。

产品优先接入讯飞语音云服务，提供实时全局听写（可向任意活动窗口输入）和音频文件转写两大核心功能，同时具备智能后处理能力（自动去除语气词、标点补全、语音命令控制）。后期支持多引擎切换，满足不同场景下对隐私与精度的差异化需求。

---

## Problem Statement

**当前痛点**：现有语音听写工具（如 Typeless）对中文的识别效果欠佳，尤其在中文普通话、方言及技术词汇场景下准确率不足；同时商业订阅费用较高，且数据上传至境外服务器存在隐私顾虑。

**解决方案**：构建一款以国内云 ASR（讯飞为主）为底层引擎、Electron 跨平台部署的语音听写工具，支持全局听写、文件转写、智能文本后处理及语音命令控制。

**业务价值**：替代手动打字，预期将日常文字输入效率提升 3-5 倍；节省商业订阅成本；数据留存国内，满足隐私要求。

## Success Metrics

**主要 KPIs：**
- **打字替代率**：上线后 1 个月内，日常输入场景中 80%+ 通过语音完成
- **中文识别准确率**：实时听写准确率 ≥ 95%（标准普通话场景）
- **延迟**：实时听写端到端延迟 ≤ 2 秒
- **用户留存**：团队成员每周活跃使用天数 ≥ 4 天

**验证方式**：本地日志统计语音输入字数 vs 键盘输入字数比例；定期人工抽检转写准确率。

## User Stories & Acceptance Criteria

### Story 1：全局语音听写

**As a** 知识工作者
**I want to** 按下快捷键后对任意活动窗口进行语音输入
**So that** 无需切换应用，直接用语音替代键盘在任何场景下输入文字

**Acceptance Criteria:**
- [ ] 用户可自定义激活快捷键（默认 `Alt+Space` 或可配置）
- [ ] 激活后麦克风图标显示在系统托盘/悬浮窗，给出录音状态反馈
- [ ] 语音识别结果实时（≤2s 延迟）输入到当前光标所在位置
- [ ] 停止说话超过 1.5 秒自动结束本次输入
- [ ] 识别结果自动去除语气词（嗯、啊、就是说等）并补全标点

### Story 2：音频文件转写

**As a** 用户
**I want to** 上传会议/讲座录音文件，获得文字稿
**So that** 快速整理会议纪要和演讲内容，节省手动记录时间

**Acceptance Criteria:**
- [ ] 支持拖拽或选择 mp3/mp4/wav/m4a 等常见格式文件
- [ ] 显示转写进度条，完成后可一键复制或导出为 .txt/.md 文件
- [ ] 支持长音频（30分钟以上）分段处理，避免超时
- [ ] 转写结果按时间戳分段展示

### Story 3：语音命令控制

**As a** 用户
**I want to** 通过语音指令控制文本编辑操作
**So that** 不需要用鼠标或键盘进行基础编辑，完全保持语音工作流

**Acceptance Criteria:**
- [ ] 支持「换行」「删除」「撤销」「句号」「逗号」等基础语音命令
- [ ] 命令词可配置（支持自定义添加/修改）
- [ ] 命令词不会被当作普通文字输入

### Story 4：个性化词典

**As a** 用户
**I want to** 添加专业术语到自定义词典
**So that** 技术词汇、产品名称等能被准确识别，减少手动纠错

**Acceptance Criteria:**
- [ ] 提供词典管理界面，支持增删改查
- [ ] 词典内容在识别时优先匹配
- [ ] 支持导入/导出 csv 格式词典

## Functional Requirements

### 核心功能

**功能 1：全局语音听写**
- **描述**：系统级热键监听，激活后捕获麦克风输入，实时将语音流发送至讯飞 ASR API，将返回文字注入当前焦点窗口
- **用户流程**：按快捷键 → 出现录音指示 → 用户说话 → 文字实时出现在光标处 → 停顿超时或再次按键停止
- **边界情况**：无麦克风权限时提示授权；网络断开时提示并支持本地 Whisper 降级（后期）
- **错误处理**：API 调用失败时显示错误提示，不影响原有窗口内容

**功能 2：文件转写**
- **描述**：接收用户上传的音频/视频文件，调用讯飞长音频转写 API，返回带时间戳的文字稿
- **用户流程**：拖入文件 → 选择语言/说话人数 → 开始转写 → 查看/导出结果
- **边界情况**：文件超过 500MB 提示压缩；不支持格式提示转换

**功能 3：文本后处理**
- **描述**：对 ASR 原始输出进行智能清洗，包括去语气词、补标点、识别并执行语音命令
- **配置**：用户可在设置中开关各后处理选项

**功能 4：引擎管理**
- **描述**：MVP 使用讯飞，设置页面预留引擎切换入口（阿里云、字节火山等）
- **配置**：用户填入自己的 API Key，本地加密存储

### 超出范围（本期不做）
- 多说话人识别（Speaker Diarization）
- 实时翻译
- 移动端 App（iOS/Android 作为后期版本）
- 云端同步历史记录
- 团队协作/权限管理

## Technical Constraints

### 性能
- 实时听写端到端延迟：≤ 2 秒（网络正常）
- 文件转写：30分钟录音 ≤ 5 分钟完成
- 应用启动时间：≤ 3 秒

### 安全与隐私
- API Key 本地加密存储（使用系统 Keychain / electron-store 加密）
- 不自建后端，所有数据直接从客户端发往 ASR 服务商
- 转写历史仅存本地，不上云

### 集成
- **讯飞 WebSocket 实时 ASR**：用于全局听写，支持流式返回
- **讯飞长音频转写 REST API**：用于文件转写
- **系统级热键**：使用 `iohook` 或 Electron `globalShortcut`
- **系统托盘**：Electron `Tray` 模块

### 技术栈
- **框架**：Electron（支持 Windows 优先，后扩展 macOS）
- **前端**：React + TypeScript
- **本地存储**：electron-store（配置）+ SQLite（历史记录）
- **ASR**：讯飞 WebSocket SDK（实时）+ REST API（文件）

## MVP 范围与分期

### Phase 1：MVP（首期交付）
- 全局语音听写（快捷键激活，输入到任意窗口）
- 音频文件转写（上传 → 转写 → 导出）
- 基础文本后处理（去语气词 + 标点补全）
- 讯飞 API Key 配置界面
- 系统托盘图标 + 录音状态指示

**MVP 定义**：用户能在 Windows 上用语音替代打字完成日常文档输入和会议记录整理。

### Phase 2：增强（MVP 后）
- 语音命令控制（换行、删除、撤销等）
- 自定义词典管理
- 多 ASR 引擎切换（阿里云 / 字节火山）
- 本地 Whisper 离线降级方案
- Windows 开机自启 + 静默更新

### 未来版本
- iOS/Android 移动端
- 多说话人识别
- 实时翻译

## 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 讯飞 API 实时延迟不稳定 | 中 | 高 | 提供本地 Whisper 作为备选；设置超时重试 |
| 系统级热键与其他软件冲突 | 中 | 中 | 支持用户自定义快捷键；提供冲突检测提示 |
| Electron 性能开销大 | 低 | 中 | 做内存优化；考虑轻量化托盘模式 |
| 中文方言/口音识别差 | 中 | 中 | 明确 MVP 只保证普通话；后期扩展方言 |
| 讯飞 API Key 泄露 | 低 | 高 | 使用系统 Keychain 加密存储，不明文落盘 |
